# Daily Log — 26 June 2026

## Completed

### Metric change: accuracy → F1
- Switched the headline metric to F1 (precision + recall based) because the
  classes are imbalanced (~68% planned / ~32% impulsive)
- Confirmed this is a binary (not multilabel) problem, so a single F1 for the
  impulsive class is appropriate, reported with macro-F1, precision and recall

### Training rebuilt as an epoch-based neural network
- Replaced the Random Forest with an MLPClassifier (hidden layers 16, 8)
- Trained epoch by epoch using `partial_fit`; validation F1 measured after each epoch
- Split changed to 80% train / 10% validation / 10% test (stratified)
- Added early stopping: keep the model from the best validation epoch
  (epoch 12 here) instead of the overfitted final epoch

### Fixed the "fridge at night" problem
- Added three features: `is_essential`, `deliberation_minutes`, `on_wishlist`
- Rewrote the label rule so impulse = low deliberation + emotional trigger,
  and price is no longer part of the label
- An essential, planned (or even urgent) fridge bought at night is now
  correctly classified as planned
- Permutation importance confirms the shift: on_wishlist (0.23),
  deliberation (0.22), is_essential (0.20) dominate; price collapsed to 0.01

### Probability-based 3-level alerts
- `predict_purchase.py` now uses `predict_proba`: alert ≥ 0.80, nudge ≥ 0.50,
  otherwise fine

### Feedback loop (human-in-the-loop)
- Users can correct an assessment; the labeled example is saved to
  `Data/Real_User_Data/feedback.csv`
- `preprocess.load_dataset()` merges feedback into the next training run

### Code cleanup
- Added `config.py` (paths, features, hyperparameters) and `preprocess.py`
  (shared loading + preprocessing)
- Bundled preprocessing + model into one scikit-learn Pipeline
  (`impulse_pipeline.pkl`), removing the old separate encoder/scaler files

### Graded 4-level output
- Changed the label from binary to an ordinal level (0=none, 1=mild,
  2=moderate, 3=strong), derived from the existing impulse_score
- Prediction now shows the level + per-level probabilities; multiclass uses macro-F1

### Dataset realism + rigour
- Expanded the dataset to 10,000 rows
- Added class balancing (oversampling the minority levels, train only)
- Added Gaussian label noise so the model learns the pattern instead of
  memorizing a deterministic rule (macro-F1 dropped from ~0.90 to ~0.67 — an
  honest number, confirmed by cross-validation)
- Gave price a mild non-essential role; decoupled deliberation from on_wishlist
  (overlapping lognormal); log-transformed deliberation_minutes
- Added compare_models.py: 5-fold cross-validation vs majority + logistic baselines

## Notes
- Final test macro-F1 ≈ 0.67 (weighted-F1 0.68, accuracy 0.68) on synthetic data
- Cross-validation: neural net 0.67 vs logistic 0.66 vs majority baseline 0.12 —
  the model clearly learns, but does not beat a simple linear model on this data
- The synthetic labels are rule-based, so real-world performance will differ —
  the feedback loop is the path to real data
