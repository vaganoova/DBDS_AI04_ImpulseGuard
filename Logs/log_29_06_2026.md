# Daily Log — 29 June 2026

Focus: turning the trained model into usable products (Telegram bot, REST API,
Chrome extension), closing the feedback loop on every channel, and fixing the
bugs found along the way.

## Completed

### Telegram bot — brought up to the new model
- Updated `telegram_bot.py` (originally built for the old Random Forest) to use
  the new pipeline: loads `impulse_pipeline.pkl`, collects all 7 features, and
  returns the graded 4-level result instead of a binary yes/no.

### Telegram bot — UX overhaul
- Added a persistent "🛒 Check a purchase" button so the user never has to type
  /start again for each new check.
- Removed all Markdown formatting (no more `**` characters) — clean plain text.
- Price prompt now states euros with an example.
- Frequency changed to a 1–10 menu (buttons) instead of typing 1–20.
- Deliberation time changed to a menu of time buckets (e.g. "Less than 5
  minutes", "1–3 hours") mapped to representative minutes that respect the
  model's decision boundaries.
- Feedback buttons relabeled to consistent full level names
  (Not impulsive / Mildly / Moderately / Strongly impulsive).

### Feedback loop closed on all channels
- Added a "Was this right?" feedback step to the Telegram bot.
- Created a shared `preprocess.save_feedback()` so the bot, CLI and API all
  write feedback identically.
- Fixed a bug where the bot and CLI wrote feedback columns in different orders
  to the same CSV (would have corrupted the data); both now use the shared helper.

### FastAPI backend
- New `Source/Backend/api.py` serving the model over HTTP:
  `POST /predict`, `POST /feedback`, `/` health check, `/docs` interactive UI.
- Reuses the same pipeline, feature list, level names and feedback mechanism.
- CORS enabled so the browser extension can call it.

### Chrome extension (MVP)
- New `Source/Extension/`: `manifest.json` (Manifest V3), `popup.html`,
  `popup.js`, `README.md`.
- Popup form collects the inputs, calls the FastAPI backend, shows the graded
  result + confidence with a color per level, and offers feedback.
- Best-effort "Detect price from page" button; manual entry as fallback.

### Shared prediction logging + channel column
- Added `preprocess.log_prediction()` used by BOTH the bot and the API, so every
  prediction is logged the same way (previously the API logged nothing).
- Added a `channel` column to `predicted_transactions.csv` to distinguish
  "telegram" vs "extension"; migrated existing rows.

### Retraining + housekeeping
- Retrained the model with collected feedback merged in
  (macro-F1 0.673 → 0.669 — negligible, as expected at 4 feedback rows; the loop
  works, the data is just too small to matter yet).
- Added to `.gitignore`: the personal study guide, `feedback.csv`, and
  `predicted_transactions.csv` (runtime/user data kept out of the repo).

## Notes
- The model artifact was also re-saved under scikit-learn 1.9.0 to match the
  local virtual environment (fixed an unpickle version warning).
- predicted_transactions.csv is a prediction LOG only and is never used as
